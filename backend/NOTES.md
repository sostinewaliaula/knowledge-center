# Important Notes for Future Development

## Migration Strategy ✅

**IMPORTANT:** For future schema changes, **always create new migration files** instead of:
- ❌ Modifying existing migration files
- ❌ Dropping and recreating entire databases

### Why?

- Preserves existing data
- Allows incremental updates
- Easy to track changes
- Can be run on production without data loss

### How?

1. Create a new file: `00X_description.sql`
2. Use `ALTER TABLE` statements for schema changes
3. Run: `node migrations/migrate.js`
4. Only affected tables are modified

### Example Pattern:

```sql
-- 005_add_new_field.sql
ALTER TABLE courses 
ADD COLUMN subtitle VARCHAR(255) NULL AFTER title;
```

See `docs/MIGRATION_BEST_PRACTICES.md` for detailed guidelines.
